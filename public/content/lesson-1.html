Lesson 1

Why AI struggles more with fixing than creating

It feels natural to ask AI to improve existing work. But in practice, starting from a clean specification is often faster and more reliable.

The intuition

  • AI is good at generating complete solutions from clear instructions.
  • Editing requires understanding messy context and preserving hidden constraints.
  • This often leads to a loop: fix → break → re-prompt → repeat.

A recurring experience

“I kept trying to get AI to fix an existing system… but every fix introduced new issues.”

Eventually, starting from a clean design with clear requirements worked immediately.

The surprising part: starting over was faster than fixing.

Key idea: AI performs best when you define the goal — not when you patch the past.

Evidence

What research and practice show

Measured outcomes

19% slower

Experienced developers using AI in real tasks took longer on average (METR, 2025).

44% accepted

Less than half of AI-generated code was usable without modification.

+27% accuracy

Providing missing context significantly improves AI editing performance.

Why this happens

  • AI optimizes for complete solutions, not minimal edits.
  • Old work contains hidden assumptions and inconsistencies.
  • Small changes can introduce unintended side effects.
  • Users enter re-prompting cycles instead of progressing.

Conclusion: Editing is a harder problem than generation — for both AI and humans.

Real task example

A real migration: Flask to Next.js

This lesson came from a real effort to migrate an older FDA labeling workflow from a monolithic Flask application into a modern Next.js interface with cleaner structure, better interaction patterns, and a more maintainable front end.

Introduction

The goal was not a cosmetic rewrite. It required translating an older research tool into a more responsive, user-friendly application architecture while preserving core workflow logic.

At first, the natural instinct was to let AI “update” the legacy implementation directly. In practice, that often created a repair cycle: one issue would be fixed, while another behavior or dependency would break.

Progress improved only after shifting from patching old artifacts to restating the requirements and redesigning the structure cleanly.

References

Evidence behind this lesson

  1. METR. Measuring the Impact of Early-2025 AI on Experienced Open-Source Developer Productivity, July 10, 2025. Reported that experienced developers using AI took 19% longer on average in a randomized controlled trial; developers expected AI to make them faster.
  2. Daye Nam et al. Prompting LLMs for Code Editing: Struggles and Remedies, arXiv:2504.20196, 2025. Found that frequent re-prompting is a marker of developer struggle and that inferring missing context improved edit correctness by 27%.
  3. Priyanshu Gupta et al. GrACE: Language Models Meet Code Edits, ESEC/FSE 2023. Conditioning models on relevant prior edits improved top-1 correctly edited code by 29% and 54% over prior methods.
  4. METR. Measuring AI Ability to Complete Long Tasks, March 19, 2025. Reported that current frontier models succeed on very short tasks but have less than 10% success on tasks taking expert humans more than around 4 hours.
What the evidence suggests

AI is not ineffective—but in complex, context-heavy tasks, repeatedly fixing existing work often adds overhead instead of reducing it.

Recommended workflow

Start with clear requirements, constraints, and examples. Let AI generate from a clean slate, then review and integrate selectively.
